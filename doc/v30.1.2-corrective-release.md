## Canonical hashing correction

Tracked provenance and epoch-contract inputs are read from exact Git blobs at
the candidate SHA. Binary and nanobenchmark files continue to use raw-byte
hashes. The verifier independently computes the same canonical tracked-input
hashes and validates every recorded epoch-contract path. CRLF-smudged working
trees must generate the same tracked-source evidence as LF working trees;
untracked, substituted, outside-repository, and non-blob inputs fail closed.
