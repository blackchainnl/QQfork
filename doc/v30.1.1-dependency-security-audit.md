# Blackcoin v30.1.1 dependency, provenance, and legacy-delta audit

Status date: 2026-07-13

This document records the evidence reviewed for the v30.1.1 dependency and
network-capability work. It is not a statement that the release is production
ready. The release remains gated on the platform builds and unresolved legacy
deltas identified below.

## Scope and immutable baselines

The audit used these immutable revisions:

* v30.1.1 integration baseline:
  `d161e279be86b3bc20a8c59d3e08e66cbacbeeaa`
* active legacy repository baseline:
  `5c1a4a5efc3886a69f01d50109c251d153ed092e`

The repositories do not have a common git object ancestor. The v30 history was
squashed before this audit. Commit presence therefore cannot be inferred from
`git merge-base`, and no legacy commit was bulk cherry-picked. Relevant changes
were compared by path and behavior because the v30 code also reserves witness
version 16 for Quantum Quasar and implements a staged shadow-ledger schedule.

The consensus-critical liboqs pin remains 0.15.0. The official
[0.15.0 release](https://github.com/open-quantum-safe/liboqs/releases/tag/0.15.0)
reports no security issues and is the current upstream release on the status
date. Its deterministic build remains limited to ML-DSA-44, disables the
OpenSSL backend and distribution dispatch, and selects generic code paths. The
complete cryptographic test and sanitizer matrix remains a separate release
gate; this dependency review does not replace it.

### Qt residual risk and reachability

Qt 5 reached the end of its standard release stream after 5.15.19. Qt reports
[CVE-2025-14575](https://www.qt.io/blog/security-advisory-untrusted-search-path-vulnerability-in-openssl)
as affecting all Qt 5 releases, with no Qt 5 fixed release. That issue concerns
the OpenSSL TLS backend on Unix-like systems. The deterministic Qt configuration
uses `-no-openssl`, `-no-schannel`, and `-no-securetransport`, and this audit
found no first-party `QSsl*` or Qt-network TLS use. The affected backend is thus
not intended to be built or reachable in the release configuration. Release
artifacts must still be inspected to confirm the configured features and linked
libraries.

The 2026 Qt SVG marker issue applies to Qt 6.7 and later, not Qt 5.15. The
deterministic build also does not download or link the `qtsvg` module. Source SVG
assets do not establish runtime use of that module.

## Quantum Quasar P2P capability

v30.1.1 advertises `NODE_QUANTUM_QUASAR` on service bit 24. The bit communicates
that a node understands the shadow ledger and its RPC surface. It is not an
authenticated readiness vote, a consensus activation signal, or a requirement
for peer selection. `GetDesirableServiceFlags()` remains unchanged, so a legacy
peer offering `NODE_NETWORK` remains desirable and mixed-version connectivity
is not partitioned by the new advertisement.

The unit test checks the serialized service name and legacy desirability. The
`rpc_net.py` and pruned-node functional tests check local service reporting and
the exact advertised mask.

## Inherited copyright provenance

`contrib/devtools/restore_inherited_copyrights.py` compares the two immutable
trees at the same path. It restores a legacy Bitcoin Core notice only when:

1. the legacy file contains exactly one such notice;
2. the current baseline contains an inherited Blackcoin-family header; and
3. the normalized non-copyright bodies are identical or have line similarity
   of at least 0.95.

Vendored subtrees, generated/data/translation files, binaries, and patch
artifacts are excluded. The resulting manifest records 664 eligible files: 306
normalized-identical and 358 at or above 95% body similarity. It records 11
excluded files and 641 ambiguous files. Of the ambiguous files, 508 are below
the threshold and 133 lack an inherited Blackcoin-family header. Ambiguous files
are not modified by the tool.

The manifest and aggregate content hash of all 664 eligible paths are unchanged
after a second `--write` run. A separate check confirms that none of the 508
below-threshold paths acquired the inherited notice. The generated manifest is
`contrib/devtools/copyright-provenance-v30.1.1.json`.

## Remaining production gates

The following work remains before this dependency stack can be classified as
production ready:

1. Build the complete deterministic dependency graph and release artifacts on
   Linux x86-64 and arm64, Windows x86-64, and macOS x86-64 and arm64. The Qt
   update has only completed sequential patch application in this audit.
2. Verify the feature set and linked libraries of every artifact, including the
   absence of the Qt OpenSSL backend.
3. Run wallet unit and functional coverage against artifacts linked to the new
   SQLite build rather than only the developer host build.
4. Run the complete multiprocess build after rebuilding both native and target
   Cap'n Proto components; the isolated native package build alone is not a
   sufficient release test.
5. Generate an SBOM and run an advisory scan against the actual release
   artifacts. Other aging platform pins, including fontconfig, libxkbcommon,
   the X11 stack, and Boost, were not established as clean by this audit.
6. Resolve or formally defer each P1 legacy delta above with an owner, rationale,
   and focused test evidence.

Until those gates pass, the dependency/security roadmap item is a release
candidate, not complete.
