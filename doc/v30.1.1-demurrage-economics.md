# v30.1.1 demurrage economics and avoidance audit

## Consensus rule

Demurrage activates automatically on the first Final Lockout block. Before that
block, it has no effect. At and after activation, the inactivity baseline for a
value-bearing quantum output is the later of its creation height and the
activation height.

The first six months of inactivity retain 100% of nominal value. Value then
decays quadratically toward zero over the next 18 months. Until the terminal
boundary, a qualifying direct-key attestation can refresh liveness. A spend or
coinstake realizes the current calculation.

The economic rule is:

```
effective input = floor(nominal input * remaining_ppm / 1,000,000)
burned principal = nominal input - effective input
transaction fee = sum(effective inputs) - sum(outputs)
```

Burned principal is destroyed. It is not a transaction fee, block subsidy,
staking reward, treasury transfer, shadow-pool credit, or claim reimbursement.
For a coinstake, the maximum output is effective principal plus the ordinary PoS
subsidy and explicit fees. The nominal-minus-effective difference cannot be
reclaimed by the staker.

At zero remaining value, the nominal UTXO remains in the UTXO set but is
consensus-unspendable and has zero effective/circulating value. A later
attestation cannot resurrect a terminal output. The sweep RPC skips terminal
outputs because no owner transaction can consume them, and an owner is never
asked to pay a fee to clean up value that has already reached zero. A future
physical-pruning mechanism would have to be automatic, deterministic,
reorg-safe, and free; it is not part of v30.1.1 and cannot change the burn.

## Exemptions and activity paths

| Path | Treatment | Avoidance finding |
| --- | --- | --- |
| Direct quantum migration output | Decays after inactivity; signed attestation can refresh before the terminal boundary | Intended liveness path; signature, controlling key, target UTXO, replay anchor, predecessor record, and chain domain are authenticated |
| Tiered quantum output | Same effective-principal calculation | A transition can return only effective principal; accrued decay cannot be carried into the new tiered output |
| Quantum cold-stake output | Subject to inactivity demurrage | Delegation alone is not exempt. A successful coinstake realizes decay, returns only effective principal plus ordinary reward, and recreates the output at a fresh height |
| Historical EUTXO-shaped output | Tracked by the demurrage evaluator, but independently frozen by v30.1.1 post-Gold-Rush consensus | No supported activity path exists: v15 outputs and spends are rejected from Migration onward because v15 lacks quantum ownership authorization. Metadata and decode/verify surfaces are inspection-only |
| Configured exempt script | Full nominal value | Explicit consensus allowlist only. Mainnet currently configures no entries; adding one is a consensus/economic change and requires separate review |
| Legacy/non-quantum output after Final Lockout | Demurrage evaluator labels it non-quantum, but Final consensus rejects spending it | Not a shelter: protected quantum spends cannot downgrade into legacy outputs, and legacy value cannot be spent after Final Lockout |
| Move to a fresh quantum address | Existing output is first valued at effective principal | Intended activity path; any accrued difference is burned before the new output receives a fresh height |
| Late attestation before terminal boundary | Refreshes the direct key's activity state | Intended proof-of-liveness behavior; no value is transferred and the attestation must be valid and continuous before terminal expiry |
| Attestation at or after terminal boundary | Cannot restore terminal coin | Coverage-epoch rule prevents resurrection |

## Accounting surfaces

`CheckTxInputs` calculates explicit fees from effective inputs. Block validation
uses those fees for both PoW and PoS reward limits. Coinstake validation also
computes `nActualStakeReward` relative to effective principal and requires the
stake-reward split to return only effective principal. This closes both direct
redistribution routes.

`getcirculatingsupply` reports nominal UTXO value separately from next-block
effective circulating value, scheduled decayed value, and Final-locked legacy
value. `getdemurragewalletinfo` labels unspent decay as `burned_if_spent_amount`;
the burn becomes realized only when a valid spend consumes the output. Explorer
and index APIs must preserve the same nominal/effective/realized distinction.

## Required regression evidence

- curve, rounding, grace, decay, and zero boundaries;
- activation and non-retroactive baseline;
- explicit fee equals effective inputs minus outputs, excluding burned value;
- PoS output cannot include burned principal;
- direct, tiered, cold-stake, frozen-v15, configured-exemption, and Final legacy paths;
- late-attestation recovery before terminal and non-resurrection at terminal;
- rejection of an owner cleanup spend even when a separate live quantum input
  offers to pay the transaction fee;
- attestation authentication, continuity, replay, restart, and reorg behavior;
- wallet sweep and supply reporting; and
- lifecycle reindex/replay with identical effective results.
